一、评测框架：六级维度·百分制

采用"自上而下"（HR筛选逻辑：匹配→可证→可见）与"自下而上"（整合HR实操、学术基准、公平性框架、已有工具维度）相结合的方法，设计了兼顾简历质量评估和大模型诊断能力评估的六级维度框架。

维度一：简历内容质量评估（30分）

评估简历本身的信息质量和表达水平，是评测的核心。

1.1 经历描述的STAR/量化程度（12分）

评估经历是否遵循STAR法则（情境-任务-行动-结果），以及是否包含可量化成果数据。

评分等级标准描述分值
优秀每段经历均用STAR结构撰写，且有具体量化数据（如"效率提升30%"）10-12
良好大部分经历有STAR结构，部分有量化数据7-9
一般有经历描述但缺乏STAR结构和量化4-6
较差仅有职责罗列，无成果描述0-3

1.2 内容相关性与精准度（10分）

评估简历内容与目标岗位的匹配程度——是否围绕岗位JD提炼关键词并逐一"对证"。

评分等级标准描述分值
优秀内容完全围绕目标岗位，关键词覆盖率高，核心卖点突出9-10
良好大部分内容与岗位相关，有一定针对性6-8
一般内容与岗位部分相关，有通用化倾向3-5
较差内容与岗位关联度低，明显是"万能简历"0-2

1.3 信息密度与语言质量（8分）

高质量简历应高信息密度、低形容词堆砌。李开复评分法：数数据量 vs 形容词量。

评分等级标准描述分值
优秀语言精炼、信息密度高、无冗余，数据/形容词比高7-8
良好语言流畅，有一定信息量5-6
一般有套话空话，信息密度一般3-4
较差大量形容词堆砌，缺乏实质信息0-2

维度二：结构与格式规范性（15分）

📋 结构完整性（5分）

是否包含完整必要信息：教育背景、工作/项目经历、技能、联系方式。经历按倒序排列。

📐 格式规范与可读性（5分）

字体一致性、间距、边距、排版整洁度。校招简历超过2页即可能被淘汰。

🗂️ 信息组织逻辑（5分）

信息分类是否清晰，最重要的信息是否在简历前1/3处，类别排序是否符合行业惯例。

维度三：岗位匹配度评估（20分）

🎓 硬性条件匹配（8分）

学历、专业、工作年限、必备技能证书等"硬性条件"是否满足岗位要求——第一道筛选门槛。

🔑 技能关键词覆盖率（7分）

简历中包含JD核心关键词3个以上的候选人，获得面试邀请的概率是未包含者的 4.2倍。

💼 经验相关性（5分）

过往经历与目标岗位的行业相关性、职责匹配度、企业规模匹配等。

维度四：诊断与优化效果评估（20分）

评估大模型诊断能力——能否准确识别简历问题并给出有效优化建议。

🔍 问题识别准确率（7分）

能否准确识别：内容空洞、缺乏量化、结构混乱、格式错误、关键词缺失、岗位不匹配。

💡 优化建议针对性（7分）

建议是否具体可执行，而非模板化。如"将'负责项目管理'改为'主导XX项目，提前2周交付，满意度95%'"。

📈 优化后提升度（6分）

对比优化前后在内容质量维度的提升幅度，可用ROUGE、BERTScore等指标辅助衡量。

维度五：公平性与偏见检测（10分）

⚖️ 人口统计学偏见（5分）

模型是否对不同性别、年龄、地域、种族群体给出系统性差异评分。可借鉴FAIRE基准检测。

🙈 非职务信息干扰（5分）

对比"匿名化简历"与"完整简历"的评分一致性，评估非职务相关信息的影响。

维度六：模型基础能力评估（5分）

🏷️ 信息抽取准确性（2分）

能否准确从简历中抽取教育背景、工作经历、技能等结构化信息。

🌐 跨语言/跨格式（2分）

处理不同语言（中英文等）和不同格式（PDF、Word、图片等）简历的能力。

🧠 推理与解释（1分）

评分是否有合理解释和依据，能否给出透明的评估理由。

等级划分

等级分数区间含义
优秀85-100模型诊断精准、优化有效、公平性良好
良好70-84模型表现较好，部分维度有提升空间
合格60-69基本可用，需在特定维度加强
待改进<60存在明显短板，需系统性优化

二、评测数据集来源

（一）通用简历解析与内容质量评测（维度一、二、六）

ResumeBench ⭐ 推荐首选

2,500份合成简历 · 50种模板 · 30个职业 · 5种语言

首个隐私合规的简历解析基准。采用"人在回路"生成管道，已有24个SOTA模型（含GPT-4o）的评测基线。

EMNLP 2025 多语言

CareerCorpus

302份专家标注简历 · 6个职业类别

两位领域专家独立标注，标注质量高。适合作为黄金标准测试集。标注者间一致性：Apparel r=0.89，Finance r=0.68。

CC-BY-4.0 专家标注

Resume Training Dataset

22,855份简历样本

包含用户与AI助手的简历修改对话数据，适用于诊断与优化效果评估。

Hugging Face 对话数据

ResumeAtlas

13,389份记录 · 43个类别

大规模简历分类数据集，适用于内容质量评估和模型基础能力评测。

arXiv 分类

（二）岗位匹配与检索评测（维度三）

RJDB ⭐ 推荐

50,000+组三元组

"职位描述-匹配简历-不匹配简历"三元组。基于技能-职业图谱生成，学生模型可接近GPT-4性能。

GitHub开源 匹配评测

PJB

6个行业 · 近200,000份简历

面向推理的人-职检索评测。不仅评测"谁得分更高"，更诊断"系统在哪里失败、为什么失败"。

arXiv 推理诊断

RecruitmentBenchmark-2025

18,500份简历 · 2,100份JD

覆盖多行业的招聘匹配评测基准。

IEEE

（三）公平性与偏见评测（维度五）

FAIRE ⭐ 推荐

种族和性别偏见测试

通过直接评分和排名两种方法，测量微调种族或性别标识后的模型表现变化。覆盖多行业，可直接复现。

GitHub开源 公平性

PopResume

60,800份简历 · 5个职业

面向因果公平性审计。采用路径特定效应（PSE）评估，能区分"业务必需"与"歧视性"差异。

arXiv 因果审计

FairCV（中文）⭐

中文互联网求职简历

针对性别、年龄、地域等多维度偏见的大模型评测。中文语境，面向国内大厂AI招聘场景。

GitHub开源 中文

JobResQA

581个QA对 · 5种语言含中文

多语言问答基准，通过占位符控制人口统计属性，同时评测跨语言能力。

GitHub开源 多语言

数据集选用建议

评测维度推荐数据集理由
内容质量CareerCorpus有专家双标注，可作为Ground Truth
结构格式ResumeBench含50种模板，可测结构化理解
岗位匹配RJDB 或 PJB规模大，覆盖多行业
诊断优化Resume Training Dataset含简历修改对话

实施路径

  1. 先用 CareerCorpus（302份）做小规模验证 — 有专家标注，成本低，快速验证一致性
  2. 再用 ResumeBench（2,500份）做全面评测 — 规模适中，已有SOTA基线可对比
  3. 公平性专项用 FAIRE + FairCV — 覆盖中英文不同场景
  4. 岗位匹配专项用 RJDB 或 PJB — 需深度诊断推理能力时优先PJB

三、评估体系：三层漏斗

评估体系的核心不是"能不能打分"，而是"分数是否可信、可复现、可对标"。

L1 · 自动化客观评测层（40%权重） 跑结构化指标：准确率、召回率、格式合规 → 定量数据
L2 · LLM-as-Judge 主观评测层（40%权重） 用强模型（GPT-4o/Claude）按细则打分 → 定量分数
L3 · 人工专家抽检层（20%权重） HR专家盲审，校准机器评分偏差 → 定性校准

最终分数 = L1×0.4 + L2×0.4 + L3×0.2，并输出置信区间

四、核心指标体系

2.1 自动指标（L1）

维度自动计算指标算法/工具
内容质量-量化程度简历中数字的密度（个/百字）正则提取
岗位匹配-关键词覆盖简历与JD的Jaccard相似度分词+TF-IDF
格式规范性是否包含三模块；页数≤2；日期格式统一规则校验
信息抽取准确性模型抽取实体与Ground Truth的F1值精准匹配/Levenshtein

算法公式：维度一量化得分 = min(数字密度 / 基准密度(1.5个/百字), 1) × 该维度满分

2.2 LLM-as-Judge 指标（L2）

  • 输入：原始简历 + 目标岗位JD + 模型诊断报告/优化建议
  • 输出：按6维度细则，输出0-100分及理由
  • 关键操作：同一样本跑3次（温度=0），取中位数，计算标准差作为置信度

2.3 人工抽检指标（L3）

抽检10%-20%样本，由3位HR背靠背打分，计算组内相关系数（ICC）。若ICC < 0.7则需调整L2评估提示词。

基线对标体系

基线类型模型/方法用途
随机基线随机生成诊断意见（或规则模板）看模型是否"至少不是废话"
SOTA基线GPT-4o / Claude 3.7 Sonnet看距离顶级差多远
人类基线3年+HR专家打分（L3数据）看能否达到"类人水平"

五、评估执行流程（SOP）

分4个阶段，总周期约 2-3周：

Phase 1：配置与预热（2天）

  1. 从评测集抽取20份"校准集"（覆盖好/中/差简历）
  2. 3位HR专家独立打分 → 得到人类基准分（Gold Standard）
  3. 用这20份调试LLM-as-Judge提示词，直到ICC≥0.75

Phase 2：自动化批量跑分（3天）

  1. 评测集全量输入模型 → 产出诊断报告
  2. 同时跑GPT-4o作为SOTA基线（成本可控跑500份子集）
  3. 跑通所有L1自动指标（正则、TF-IDF、实体F1）

Phase 3：LLM裁判打分（3天）

  1. 模型诊断报告 + 原始简历 + JD → 灌入"裁判模型"
  2. 每个样本跑3次取中位数，产出6维分数
  3. 裁判模型也跑人工校准集的20份，校验一致性

Phase 4：人工抽检与报告生成（3天）

  1. 随机抽检10%（250份），HR盲审打分
  2. 计算L1/L2/L3加权总分及各维度子分数雷达图
  3. 输出《模型评估报告》：强项/弱项/改进建议

六、可信度校验机制

这是评估体系的灵魂，否则结果无人信服：

校验项合格阈值若不达标
人机一致性（ICC）> 0.75调整L2裁判提示词，重新标注
裁判自洽性（3次标准差）< 3分提示词不清晰，需细化评分锚点
维度区分度（方差）> 10若分数集中在80-85，说明评分细则失效
对抗样本检测明显错误简历得分显著低于正常简历若未检出，说明敏感度不够
公平性校验（FAIRE）属性替换后分数波动 < 2%提示词增加"禁止考虑性别/年龄"指令

踩坑提醒

  • 裁判模型不能既当运动员又当裁判员 — 如果你的模型是微调Llama，裁判用GPT-4o；如果本身就是GPT-4o，裁判换Claude
  • 岗位JD缺失处理 — 部分公开数据集没有JD时，暂停维度三（岗位匹配），只评测内容质量和格式，单独报告
  • 中文评测特别注意 — 英文ResumeBench需额外加FairCV做中文专项；中英分开报告，不混算总分
